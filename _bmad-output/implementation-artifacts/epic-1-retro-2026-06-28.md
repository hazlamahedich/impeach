## What Went Well

1. **Mechanical discipline over convention.** Murat's 4 anti-hollow teeth (no `.skip/.only/.todo`, assertion floor, RED observability, real `@iip/<pkg>` imports) became machine-enforced CI gates, not eyeball-enforced hopes. Every story could prove its tests weren't hollow.

2. **Party Mode adversarial review BEFORE dev.** Stories 1.1 (6-agent roundtable), 1.4 (6 amendments), 1.5 (8 amendments) caught structural issues pre-implementation. Both 1.4 and 1.5 executed clean, single-pass — the strongest predictor of clean execution in the data.

3. **Enforcement Boundary Tables.** Every story from 1.4 onward declared what it ENFORCES and what it DOES NOT ENFORCE. This prevented false claims about the spine's completeness and set the intellectual honesty standard (INV-001 labeled yellow, not green).

4. **Previous Story Intelligence.** Each story explicitly carried forward prior stories' pins, conventions, and gotchas. Near-zero regression of Story 1.1 mandates in later stories.

5. **Deterministic structural stub over runtime throw (1.4 Amendment C).** Stronger than a throw — prevents accidental GREEN from changed error messages and provides a testable foundation for 2.1.

6. **"Proven load-bearing" validation pattern (1.10).** Mutate the linter → watch it fail → restore → green. Validates tests aren't vacuous.

7. **Canonical error handling push.** Reviewers consistently enforced `Error`→`AppError`, `as`→`instanceof`/validation (1.6, 1.11), establishing the project standard early.

---

## What Didn't Go Well / Challenges

1. **Story 1.3 was a silent scope sink.** Originally draft-blocked, it absorbed P0-P3 (defamation definition, citation RED test, CI wiring, boot runner, ordering invariant) that the epic plan didn't assign to it. It became the choke point. The Foundation Action Plan recovered it, but the plan didn't predict the weight. **Root cause:** implicit prerequisites on prior stories weren't enumerated before 1.3 went to draft.

2. **Pre-authored ATDD tests were defective in 3 of 12 stories.** Worst case: 1.9 with 7 critical defects (ESM/CJS, `require` in module, `execa`, Server Component in jsdom, `.skip`, wrong API, ROOT path). Dev time went into rewriting test scaffolds instead of implementing against them. **Root cause:** scaffolds assumed non-trivial test environments (jsdom, React, ESM) without running the code.

3. **macOS AppleDouble `._*` files recurred across 4 stories** (1.1, 1.2, 1.3, 1.4). External exFAT drive creates resource forks that break ESLint, Vitest, and Docker build context. Each story re-solved it independently. **Root cause:** environment quirk not fixed at the project level early.

4. **Turbo self-recursion on root `.` workspace member.** Latent bug from 1.1, deferred in 1.7 as "pre-existing," finally fixed in 1.8 via `--filter=!impeachment-watch`. Surfaced when lockfile changes invalidated the build cache. **Root cause:** root package is a `.` workspace member that delegates to turbo, recursing into itself.

5. **Render-gate stub is dangerously comfortable.** The structural stub passes the contract test GREEN-by-construction, which looks like the invariant is enforced. INV-001 is yellow. Real validation (substring, trust-tier, corroboration) doesn't exist yet. **Risk:** false confidence carried into Epic 2.

6. **RED↔GREEN contract-test CI plumbing churned across 4 stories** (1.3, 1.4, 1.11, 1.12). Expected-RED marker → structural-stub GREEN → INV-001 red→yellow → split `.only`/`.skip` steps. The transition semantics were revisited repeatedly.

---

## Key Insights

- **Party Mode pre-dev review is the highest-leverage process investment.** It correlates directly with clean single-pass execution. Formalize it as the default for boundary/contract/new-env stories.
- **The epic plan under-counted foundation prerequisites.** Story 1.3's draft-block was recoverable but predictable. Implicit dependencies must be surfaced during epic planning, not during story execution.
- **ATDD scaffolds are specifications, not runnable code**, for any non-trivial test environment. The "Task 0: rewrite from spec" pattern (adopted ad hoc in 1.8, 1.9) should be formalized.
- **Intellectual honesty about stubs is a deliverable.** Labeling INV-001 yellow (not green) and shipping Enforcement Boundary Tables is more valuable than claiming completeness.
